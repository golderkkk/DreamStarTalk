# AI Chat Studio vs SillyTavern — 功能对比与优化建议

> 更新日期: 2026-07-06

---

## 一、对比总览

| 维度 | AI Chat Studio | SillyTavern | 差距 |
|------|---------------|-------------|------|
| 核心对话 | ✅ | ✅ | 基本持平 |
| 角色卡系统 | ✅ TavernCard V2 | ✅ V2 + V3 | 缺少 V3 |
| 多角色对话 | 🟡 基础支持 | ✅ 完善 | 缺少分组管理 |
| 上下文管理 | 🟡 Token 截断 | ✅ Lorebook 动态注入 | **关键差距** |
| 消息操作 | 🟡 仅删除 | ✅ 编辑/重生成/滑动 | **需要补充** |
| UI/UX | ✅ Aurora 设计 | 🟡 基础 Web | AI Chat Studio 更优 |

---

## 二、SillyTavern 核心功能对比

### 2.1 Lorebook / World Info（最关键差距）

**ST 功能**: 根据关键词自动在上下文中注入相关世界设定条目。当用户消息或 AI 回复中出现触发词时，对应的 Lore 条目会自动追加到系统提示中。

**当前状态**: AI Chat Studio 有世界观模块，可以手动选择场景和 NPC 注入上下文，但**不支持基于关键词的自动触发**。所有世界观信息都是一次性注入，无法按需动态加载。

**建议**: 在世界观条目（场景/NPC/自定义条目）中增加「触发关键词」字段，ChatService 发送消息前扫描上下文，自动注入匹配的条目。

**优先级**: 🔴 高

---

### 2.2 Author's Note

**ST 功能**: 用户自定义的一段文字，注入到对话上下文的特定位置（通常是最后一条消息附近），用于临时调整 AI 行为（如"今天的语气温柔一些"）。

**当前状态**: 部分支持。对话风格设置为全局生效，但缺少**当前对话专属的临时指令**。`customSystemPrompt` 在 `ConversationSettings` 中已定义但无 UI 入口。

**建议**: 在聊天页增加「作者笔记」输入区域（可折叠），用户输入的临时指令会被注入到最新用户消息之前。

**优先级**: 🟡 中

---

### 2.3 消息重生成 (Swipe/Regenerate)

**ST 功能**: 用户向左/右滑动可以查看 AI 对同一条消息生成的多个候选回复，选择最满意的一条。

**当前状态**: ❌ 未实现。AI 每条消息只有一个回复。

**建议**: 在消息气泡下方添加「重生成」按钮，点击后可生成替代回复。保存多个候选回复，用户可左右滑动切换。

**优先级**: 🔴 高

---

### 2.4 消息编辑

**ST 功能**: 用户可以编辑自己或 AI 的已发送消息，编辑后重新生成后续对话。

**当前状态**: ❌ 未实现。消息不可编辑。

**建议**: 长按消息弹出菜单中增加「编辑」选项，编辑后自动删除后续消息并重新生成。

**优先级**: 🟡 中

---

### 2.5 分支对话 (Checkpoints)

**ST 功能**: 可以在任意位置创建对话分支，支持回到历史节点尝试不同走向。

**当前状态**: ❌ 未实现。

**建议**: 添加对话快照功能，用户可以保存当前对话状态，随时回到某个快照点继续。

**优先级**: 🟢 低

---

### 2.6 Persona（用户角色卡）

**ST 功能**: 用户可以创建多个"人格"（Persona），每个对话可绑定不同的 Persona，AI 会根据 Persona 信息调整对自己的称呼和互动方式。

**当前状态**: 🟡 部分支持。创建对话时有"主角设置"步骤，可以填写称呼、性格等，但：
- 主角信息无法保存为模板复用
- 无法为已有对话更换主角
- 缺少 Persona 管理页面

**建议**: 将主角设置提取为独立的 Persona 管理系统，支持创建/编辑/删除 Persona，对话可随时切换绑定的 Persona。

**优先级**: 🟡 中

---

### 2.7 对话导出

**ST 功能**: 导出对话为 JSON/TXT，可包含角色卡信息。支持多种格式。

**当前状态**: ❌ 未实现。仅可导出角色卡。

**建议**: 添加对话导出功能（JSON/TXT），选项包含/排除角色卡信息。

**优先级**: 🟡 中

---

### 2.8 角色卡 V3 支持

**ST 功能**: TavernCard V3 格式，除了基本角色信息外还支持 Character Book（角色专属 Lorebook）。

**当前状态**: 🟡 仅支持 TavernCard V2 PNG 导入。

**建议**: 添加 V3 格式的导入/导出支持，支持 Character Book 字段。

**优先级**: 🟢 低

---

### 2.9 Markdown 渲染

**ST 功能**: 消息中支持完整的 Markdown 渲染（标题、列表、代码块、表格等）。

**当前状态**: 🟡 仅支持 `*动作*`、`"对话"`、`**粗体**` 三种格式。

**建议**: 使用 `flutter_markdown` 替换自定义富文本渲染。

**优先级**: 🟡 中

---

### 2.10 对话分组/标签

**ST 功能**: 对话可以加入分组或打标签，方便管理大量对话。

**当前状态**: ❌ 未实现。对话仅按时间排序。

**建议**: 添加对话标签系统，对话列表可按标签筛选。

**优先级**: 🟢 低

---

### 2.11 Token 计数器

**ST 功能**: 实时显示当前对话的 Token 使用情况。

**当前状态**: ❌ 未实现。

**建议**: 在聊天页底部或顶部显示 Token 用量（已用/总计）。

**优先级**: 🟢 低

---

### 2.12 快捷键支持

**ST 功能**: 丰富的键盘快捷键（Ctrl+Enter 发送、Ctrl+Up 编辑上条消息、Ctrl+S 保存等）。

**当前状态**: ❌ 未实现。仅触摸操作。

**建议**: 桌面端添加常用快捷键。

**优先级**: 🟢 低

---

### 2.13 变量/宏系统

**ST 功能**: `{{char}}`, `{{user}}`, `{{persona}}` 等变量，在系统提示词中自动替换。

**当前状态**: ❌ 未实现。所有提示词为静态文本。

**建议**: 在系统提示词中支持 `{{charName}}`, `{{userName}}`, `{{worldName}}` 等变量替换。

**优先级**: 🟢 低

---

### 2.14 对话书签

**ST 功能**: 长对话中可以添加书签，快速跳转到重要节点。

**当前状态**: ❌ 未实现。

**建议**: 消息长按菜单中增加「添加书签」，书签栏快速跳转。

**优先级**: 🟢 低

---

### 2.15 聊天背景

**ST 功能**: 可自定义聊天页背景图。

**当前状态**: ❌ 未实现。纯色深色背景。

**建议**: 设置中增加聊天背景自定义功能。

**优先级**: 🟢 低

---

## 三、当前 UI/UX 优势（vs SillyTavern）

| 优势 | 说明 |
|------|------|
| Aurora 设计系统 | 5 级背景层次 + 极光紫罗兰主色 + 系统化 Token |
| 移动端原生体验 | Flutter 原生渲染，流畅度远超 ST 的 Web 界面 |
| PC 双端自适应 | LayoutBuilder 自动切换移动端/桌面端布局 |
| 富文本消息渲染 | 原生支持 `*动作*` 斜体 + `"对话"` + `**粗体**` |
| 快捷回复 | 上下文感知的智能回复建议 |
| 交错入场动画 | 列表项延迟渐入动画 |
| 删除撤销 | 3 秒 SnackBar 撤销机制 |
| 内置预设 | 20 角色 + 13 世界观模板 |

## 四、优化路线图建议

### 第一阶段（近期 — 1-2 周）

| 功能 | 影响 |
|------|------|
| Lorebook 关键词触发 | 世界观动态注入，大幅提升对话沉浸感 |
| 消息重生成 (Regenerate) | 用户可生成多个备选回复 |
| Author's Note UI | 临时指令快速调整 AI 行为 |

### 第二阶段（中期 — 2-4 周）

| 功能 | 影响 |
|------|------|
| Persona 管理 | 主角模板复用 |
| 消息编辑 | 修正错误后继续对话 |
| 对话导出 | 数据迁移和分享 |
| Markdown 渲染增强 | 更丰富的消息格式 |

### 第三阶段（远期 — 1-2 月）

| 功能 | 影响 |
|------|------|
| 分支对话 (Checkpoints) | 时间线探索 |
| Token 计数器 | 用量可视化 |
| 对话标签/分组 | 大量对话管理 |
| 快捷键 | 桌面端效率 |
